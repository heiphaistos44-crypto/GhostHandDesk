Conçu pour la sécurité

🔐

Chiffrement E2E E2E

X25519 ECDH pour l'échange de clés + AES-256-GCM pour le flux vidéo. Aucune donnée en clair ne quitte votre machine.

🌐

WebRTC P2P

Connexion directe entre machines via WebRTC. Le serveur de signaling ne voit que les métadonnées de connexion.

🔑

Authentification

Mot de passe dérivé par PBKDF2-SHA256 (100k itérations). Device ID 128-bit cryptographique.

📋

Presse-papiers & Fichiers

Synchronisation du presse-papiers et transfert de fichiers entre les machines chiffrés de bout en bout.

💬

Chat intégré

Messagerie en cours de session sans quitter l'interface. Tout chiffré.

🆓

Open Source · Gratuit

Code source disponible sur GitHub. Aucun abonnement, aucune télémétrie, aucun compte requis.

Comment ça marche

1

L'hôte lance GhostHandDesk

Un Device ID unique est généré. L'hôte partage son ID et un mot de passe au visiteur.

2

Le visiteur se connecte

Il entre l'ID et le mot de passe. La négociation WebRTC s'effectue via le serveur de signaling ghd.heiphaistos.org.

3

Connexion directe chiffrée

Une fois le tunnel WebRTC établi, le serveur est hors-jeu. Le flux vidéo et audio est direct, chiffré E2E.

4

Contrôle total

Souris, clavier, presse-papiers, fichiers — tout fonctionne comme si vous étiez physiquement devant la machine.

Stack technique

Tauri v2 Rust Vue 3 TypeScript Go (signaling server) WebRTC X25519 ECDH AES-256-GCM WebSocket PBKDF2-SHA256
